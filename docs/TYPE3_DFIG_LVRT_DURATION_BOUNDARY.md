# Type-3 DFIG 固定 0.10 Ω 故障持续时间响应边界扫描

## 任务范围

本轮扫描固定 `Fault ON Resistance = 0.10 ohm`、`Fault type = ABC-to-ground`、`Fault start = 2.0 s`、`Fault OFF Resistance = 1.0E6 ohm`，只改变故障持续时间。PSCAD GUI 参数修改、保存、Clean、Build 和 Run 均由用户手动完成；本轮未使用 Computer Use，Codex 未打开或操作 PSCAD GUI，未直接编辑 `.pscx/.pslx/XML`，未修改 Type3WTG_Lib、Type-3 控制器、Dblk/Vwind 支架、变压器、旧 IBR、P/Q/V 表计、保护、MATLAB 或连锁故障逻辑。

本轮没有实现 LVRT 脱网逻辑，没有接入或操作 `BRK_DFIG`，因此结果只能说明在未脱网的 Type-3 动态模型中，不同故障持续时间对应的电压、DC-link、chopper、Crowbar、RSC/GSC 电流、功率和机械转速响应。它不能表述为风场已经脱网、必然脱网或论文完整事故链复现。

统计窗口按场景持续时间动态定义：

- `pre_fault`: `1.8 s <= t < 2.0 s`
- `fault`: `2.0 s <= t <= 2.0 s + fault_duration`
- `early_post_fault`: 故障清除后 0.5 s 内
- `late_post_fault`: `max(7.0 s, fault_clear_time + 0.5 s) <= t <= output_end_time`

## 动态趋势

1. 随着故障持续时间从 `0.15 s` 增至 `1.50 s`，`VIBR1_2` 最低值从约 `0.514785 pu` 下降到约 `0.353510 pu`，电压跌落更深且恢复更晚。
2. 六个场景都在 8 s 窗口内恢复到 `0.90 pu` 以上；最长 `1.50 s` 故障的恢复时间约为 `5.63985 s`。
3. `Edc_pu` 故障窗峰值在六个 0.10 Ω 场景中均约为 `1.227390`，`Ecap_Det` 峰值约为 `1.779061 kV candidate`，与 `Edc_pu x 1.45 kV` 近似一致。
4. `BRK_CHOP` 在所有场景中均出现短时响应，故障窗内累计约 `0.00415 s`；由于本轮没有重新进入 GUI 确认 chopper 开关逻辑方向，文档中仍称为 chopper 候选响应。
5. GUI 语义确认后的 `S1` 作为 Crowbar 开关命令状态，在所有场景中均未投入；`Crowbar current:1..3` 最大绝对值仅约 `7.66e-06`，与未导通判断一致。
6. `I_RS:1..3` 和 `Iconv:1..3` 为瞬时三相端子电流，不是 RMS。六个 0.10 Ω 场景中 RSC/GSC 最大瞬时电流分别约 `0.914968` 和 `0.967103`，明显低于 0.01 Ω 深故障恢复基准中的 RSC/GSC 峰值。
7. 后期 `PIBR1_2` 均值约为 `199 MW`，`Wpu` 回到约 `1.200`，说明未脱网模型在这些持续时间下可恢复到稳定发电状态。

## 与未来 LVRT 判据的关系

此前项目已记录工程实现边界：`Vs <= 0.20 pu` 时按 `tVRT = 0 s` 处理，即未来外部 LVRT 原型应立即脱网。这是项目实现选择，不等同于宣称论文在边界处完全无歧义。

若仅以本轮 `VIBR1_2` 的最低电压作为保守参考，并套用已审计的线性 LVRT 允许时间，`1.25 s` 和 `1.50 s` 场景会超过参考窗口，`1.00 s` 场景接近但仍在参考窗口内。这个判断只用于后续外部 LVRT 脱网逻辑设计的候选边界，不代表本轮 PSCAD 已经发生脱网，因为本轮未接入任何脱网逻辑，也没有操作 `BRK_DFIG`。

## 工程判断与限制

- 当前 0.10 Ω 故障比 0.01 Ω 深故障温和，但持续时间增加会显著延长低电压暴露时间，并推迟恢复到 0.90 pu 的时刻。
- 本轮没有发现数值稳定性、输出完整性或模型定义方面的新风险。
- 当前仍不能宣称风机脱网、LVRT 保护动作或论文连锁故障阶段已复现。
- `VIBR1_2` 可继续作为未来外部 LVRT 原型的候选 `Vs` 输入，但其 base、采样策略和计时器复归逻辑仍需在实现阶段明确。
