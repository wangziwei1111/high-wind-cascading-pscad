# Type-3 Library Dependency Scope

## Core Definition

Static read-only parsing identifies the reusable core module definition as:

`Type3_WTG_Avg`

External port:

- `AC_sys`: natural 3-phase electrical port.

Top-level parameter bindings include:

- `Dblk`
- `freq`
- `VLL_Gr` / `Vbase`
- `No_WTG` / `UN`
- `Pbase` / `Rated_MW`
- `Vwind`
- `Mrating` / `Machine_MVA`

## Definitions Expected in Library Import

When `Type3_WTG_Avg` is exported with dependents, the imported library should contain at least these project-local definitions:

| Definition | Role |

Evidence: `data/reference/pscad_case_library_evidence.json`, generated by `scripts/inspect_type3_component_definition_dependencies.py`.

## Objects Not in the Reusable Library Boundary

Do not migrate the standalone test environment:

- `V_source` / `Vsource`
- `BusPOC`
- standalone `master:source3`
- standalone `master:xfmr-3p2w`
- `Cable_1` and `Cable_3` unless deliberately reviewed as reusable collector impedance
- `POC Fault`
- `Terminal Fault`
- standalone graphs, meters, sliders, fault panels, and test controls on `Main`

Reason: these are standalone case context around the reusable Type-3 block. The intended library boundary is `Type3_WTG_Avg`, not the Type-3 case `Main` page.

## Validation Without IEEE-39 Wiring

After import into `Type3WTG_Lib`, verify only:

- Definitions branch contains the expected core/dependent definitions.
- `Type3_WTG_Avg` opens in definition view.
- `AC_sys` port exists.
- Parameters `Dblk`, `freq`, `Vbase`, `UN`, `Rated_MW`, `Vwind`, `Machine_MVA` appear.
- The library can create an instance in an empty scratch case or on a disconnected temporary page without wiring to IEEE-39.

Do not build/run IEEE-39 or edit the 3IBR network during this validation.
